hexo+github搭建个人博客
Created 2020-04-19| Updated 2020-04-19|工具
Post View:

本人程序员一枚，自从开始从事本行就心心念念的想搞一个自己的博客，怎奈有心无行动。
最近恍然一悟，最好的行动的时间永远都是当下，在不行动，永远只能羡慕别人的博客了。

话不多说，干起来！！
经过1,2的天的准备，我选择了用hexo+github 框架来搭建自己的博客。

一. hexo安装

这个直接参考其官网hexo官网操作即可，官网文档很详细。

1.1. hexo安装完成后，新建hexo项目

新建一个要存放代码的空文件夹，cmd命令进入此文件夹，执行命令：
hexo init

此时项目创建成功，项目结构如下：

1.2 主要关注以下目录及文件:

_config.yml 网站的配置信息，可以在此配置大部分的参数。

source/_posts 存放我们编写的文档的路径

themes 主题文件夹,你down的喜欢的主题文件都在此

1.3. 使用命令 hexo g 生成静态页面

命令执行后会生成public文件夹


1.4. 此时运行命令 hexo s 来查看生成的静态页面

二. 更换主题

若当前主题不是你喜欢的风格，可以更好为自己想要的风格

2.1 执行以下命令

git clone -b master https://github.com/jerryc127/hexo-theme-butterfly.git themes/Butterfly

此时在你的theme路径下会看到刚加入的主题

修改站点配置文件_config.yml,把主题修改为Butterfly

接下来执行以下命令
hexo clean // 清除旧的主题
hexo g // 生成新的主题
hexo s

对应的解决方案可以在Butterfly的Github主页看到：

npm install hexo-renderer-pug hexo-renderer-stylus

再次
hexo clean
hexo g
hexo s

开始编辑博客内容吧

  1. 创建“分类”选项
    生成“分类”页并添加tpye属性,进入博客目录。执行命令下方命令

$ hexo new page categories

categories文件夹下会有index.md这个文件，打开后默认内容是这样的：

title: categories
date: 2019-04-22 14:47:40

添加type: “categories”到内容中，添加后是这样的：

title: 分类
date: 2019-04-24 15:30:30
type: categories

保存并关闭文件。给文章添加“categories”属性打开需要添加分类的文章，为其添加categories属性。下方的categories:Hexo表示这篇文章添加到到“Hexo”这个分类。注意：一篇文章只会添加到一个分类中，如果是多个默认放到第一个分类中。

title: Hexo 添加分类及标签
date: 2017-05-26 12:12:57
categories: android

至此，成功给文章添加分类，点击首页的“分类”可以看到该分类下的所有文章。当然，只有添加了categories: xxx的文章才会被收录到首页的“分类”中。

  1. 创建“标签”选项生成“标签”页并添加tpye属性

$ hexo new page tags

在tags文件夹下，找到index.md这个文件，打开后默认内容是这样的：

title: 标签
date: 2019-04-22 14:22:08

添加type: “tags”到内容中，添加后是这样的：

保存并关闭文件。给文章添加“tags”属性,打开需要添加标签的文章，为其添加tags属性。

这些属性和属性值之间必须有一个空格，否则解析错误。

加入站点内容搜索功能步骤如下：
运行：
npm install hexo-generator-searchdb –save

在根目录的下的config.yml 页面最后添加代码。
#next search
search:
path: search.xml
field: post
format: html
limit: 10000

// 在android目录下面创建了一个名叫test的文件，标题为a
hexo new post -p android/test “a”
