Uni-app离线打包需要资源 APP离线SDK版本要与Hbuilderx版本一致

Android Studio生成打包原生工程
Hbuilderx将uni项目生成离线打包资源
App离线sdk

新建原生工程

打开Android studio新建一个空白项目,选择 Empty Activity 点击 Next

Minimum API Level 建议选19及以上,点击 Finish 完成创建

注意：App离线SDK不支持Kotlin

配置工程

基础库配置

将下载的app离线sdk里目录…\2.6.16\Android-SDK@2.6.16.80137_20200426\SDK\libs下的文件lib.5plus.base-release.aar、android-gif-drawable-release@1.2.17.aar、uniapp-release.aar和miit_mdid_1.0.10.aar拷贝到刚创建的工程的libs目录下

在build.gradle中添加资源引用

uni-app配置时需要在build.gradle中添加aaptOptions配置

应用配置

配置应用版本号

versionCode为应用的版本号（整数值），用于各应用市场的升级判断，建议与manifest.json中version -> code值一致
versionName为应用的版本名称（字符串），在系统应用管理程序中显示的版本号，建议与manifest.json中version -> name值一致。
applicationId为应用的包名，一般设置为反向域名，不建议修改。
建议将targetSdkVersion设置为25或以上。
App离线SDK minSdkVersion最低支持19，小于19在部分4.4以下机型上将无法正常使用。
